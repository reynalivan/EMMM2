# EMMM2 Changelog Style Guide

## 1. Format
Use Markdown bullet points. Bold the "Title" of the change.

```markdown
- **<Title>**: <Description of benefit>.
```

## 3. Tone
-   **Exciting**: Use active verbs (Added, Improved, Fixed).
-   **Clear**: Avoid "Refactored backend service layer". Use "Improved app stability".
-   **Concise**: One line per item usually sufficient.
