# EMMM Changelog Style Guide

## 1. Format

Use Markdown bullet points. Bold the "Title" of the change.

```markdown
- **<Title>**: <Description of benefit>.
```

## 2. Categories & Emojis

### ✨ New Features

For new capabilities.

- **Safe Mode**: Added a toggle to hide sensitive content.
- **Dark Mode**: Now supports system theme sync.

### 🔧 Improvements

For optimizations and updates.

- **Faster Scan**: Mod scanning is now 2x faster using Rust.
- **UI Polish**: Smoother animations on the Sidebar.

### 🐛 Bug Fixes

For repairs.

- **Fixed Crash**: Resolved app close when loading large skins.
- **Login Fix**: Users can now stay logged in after restart.

## 3. Tone

- **Exciting**: Use active verbs (Added, Improved, Fixed).
- **Clear**: Avoid "Refactored backend service layer". Use "Improved app stability".
- **Concise**: One line per item usually sufficient.
