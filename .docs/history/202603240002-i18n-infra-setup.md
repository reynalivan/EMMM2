# 202603240002-i18n-infra-setup

## i18n Infrastructure Setup

### Context

Requested to implement multi-language support (i18n) for the EMM2 application. This session focused on Phase 1: Infrastructure Setup, as per user's request.

### Changes

- Integrated `react-i18next` and `i18next` for internationalization.
- Created `src/lib/i18n.ts` for central i18n configuration.
- Extended `AppSettings` and `useSettings` to support language persistence and dynamic switching.
- Implemented `useLanguageRuntime` hook to synchronize UI language with settings on startup.
- Enabled and connected the language selector in the Settings -> General tab.

### Impacted Files

- `package.json` (modified)
- `src/main.tsx` (modified)
- `src/App.tsx` (modified)
- `src/types/settings.ts` (modified)
- `src/hooks/useSettings.ts` (modified)
- `src/lib/i18n.ts` (added)
- `src/features/settings/hooks/useLanguageRuntime.ts` (added)
- `src/features/settings/tabs/GeneralTab.tsx` (modified)

### Goal

The application now has a solid infrastructure for multi-language support. Users can switch between English, Bahasa Indonesia, and Chinese from the settings, and the preference is persisted.

### Impact

- UI is now ready for full text extraction (Phase 2).
- Added overhead of i18n library (~30KB gzipped).
- Immediate support for translating settings-related labels.

### Notes

- Used inline resources in `i18n.ts` for Phase 1 to minimize file creation; will move to JSON files in Phase 2.
