## Deletion Policy

1. `rg` must show no active caller outside tests/docs, or the candidate must be replaced by a canonical path.
2. Command deletion requires production registry, Specta export, permission TOML, and frontend binding audit to pass.
3. Filesystem-truth changes must preserve: FileWatcher trigger-only, Disk Reconcile source of truth, runtime-sync refresh bus, WorkspaceViewModel UI read-model.
4. Schema/domain compatibility fields remain if they are still data-state, even when the old command surface is gone.
